# Phase 5 V2 - 5.3 进度与经济闭环修复（高价值选择注入）

**阶段**: 5.3  
**优先级**: P1  
**对应问题**: F4, F5, F6, F8

---

## 1. 阶段目标

建立“成长有选择、掉落有分层、Biome 有个性”的中程循环，并把“关键选择”注入每层流程。

完成标准：

1. 护甲收益曲线在全流程有效。
2. 掉落随楼层分层，不再早期混入全部高阶池。
3. biome `lootBias` 真正影响掉落分布。
4. 升级具备属性选择而非固定加点。
5. 每层至少 1 个会改变构筑方向的高价值选择节点。

---

## 2. 问题定义

1. 线性减伤公式导致后期护甲边际价值过弱。
2. 虽有 `minFloor` 过滤机制，但内容配置分层不足。
3. `lootBias` 定义存在但未进入掉落算法。
4. 升级默认固定加 `strength`，构筑差异不足。
5. 即便有掉落和升级，玩家仍可能感知为“只是数字变大”。

## 4. 技术方案

### 4.1 护甲公式

1. 从线性减法改为：`damage * (1 - armor / (armor + K))`。
2. `K` 默认候选区间：`80~140`，通过回归确定。
3. 保留 `minDamage` 下限避免完全免伤。

### 4.2 掉落分层

1. 为 `starter/cathedral/catacomb/boss` 设置明确楼层带。
2. rare/unique 进入层级必须与 Floor 进度一致。
3. 关键事件和商店池同步分层，避免旁路绕过。

### 4.3 biome lootBias

1. `rollItemDrop` 增加可选 `slotWeightMultiplier` 输入。
2. 从 `currentBiome.lootBias` 构建权重修正表。
3. 在不破坏 determinism 的前提下调整采样概率。

### 4.4 升级属性选择

1. 将 `applyXpGain(..., statPreference)` 改为由升级面板决策。
2. 升级给 2~3 个属性选择项（可含 archetype 推荐）。
3. 同步平衡 DEX 过强问题（属性贡献拆分）。

### 4.5 高价值选择预算（Taste-T1/T3）

1. 引入“每层选择预算”概念：每层至少产出 1 个构筑分岔点。
2. 分岔点来源优先级：`升级选择 > 商店抉择 > 事件奖励 > 关键掉落`。
3. 当本层前段未命中分岔点时，后段通过商店/事件补偿，保证体验下限。

## 6. 验证方案

### 6.1 自动化

```bash
pnpm --filter @blodex/core test
pnpm --filter @blodex/game-client test
pnpm ci:check
```

关键新增测试：

1. 护甲在低中高区间均有合理收益。
2. Floor 1~5 掉落稀有度分布满足目标区间。
3. 不同 biome 下装备槽位分布存在统计差异。
4. 升级属性选择结果可追踪并影响战斗。
5. 每层至少 1 次构筑分岔点。

### 6.2 手动冒烟

1. 默认优先使用金手指快速推进 Floor 1~5，记录每层分岔点。
2. 切换不同 biome 验证偏置。
3. 选择不同升级路径（STR/DEX/INT）体验差异。
4. 记录心跳时刻（关键掉落/关键升级/关键分支）。
5. 补 1 局非金手指验证进度节奏。

1. 风险：掉落重配导致前期过穷或后期爆仓。  
   止损：引入 floor band 期望值监控。
2. 风险：升级选择导致最优解单一。  
   止损：增加属性贡献分离与弱势路径补偿。
3. 风险：分岔保障破坏随机性。  
   止损：只保障“出现一次”，不保障“给最优解”。
