# Phase 5 V2 - 5.5 体验增强重基线（心跳时刻表达层）

**阶段**: 5.5  
**优先级**: P1  
**对应来源**: 原 5.2/5.3/5.4/5.5 体验类工作

---

## 1. 阶段目标

在规则与架构地基稳定后，集中做体验表达层：把“构筑差异”转译为可感知的节奏和心跳时刻。

完成标准：

1. 移动可达性显著提升。
2. 战斗触感反馈闭环完整。
3. 地牢拓扑不再单链。
4. 氛围与可读性提升且不遮挡核心信息。
5. 每局稳定出现至少 3 次心跳时刻。

---

## 2. 范围与非目标

### 2.1 范围

1. 移动与输入：走廊宽度、clamp 容错、键鼠互操作。
2. 战斗反馈：hitstop、击退、伤害数字分层、击杀峰值反馈。
3. 拓扑策略：MST + 回边 + deterministic。
4. 心跳表达：关键掉落、关键升级、关键击杀、关键分支提示。
5. 氛围层：粒子、ambient loop、新手提示、fallback。

### 2.2 非目标

1. 不再修改核心数值单位。
2. 不再修改 Host Port 架构策略。
3. 不做大规模美术重绘。

---

## 3. 任务拆分

### PR-5.5-01 Mobility

1. `procgen` 走廊宽度参数化并扩大。
2. `MovementSystem` 容差策略升级。
3. 输入冲突规则明确（鼠标/方向键/WASD）。

### PR-5.5-02 Combat Feel

1. 新增可控 hitstop 与限频。
2. 击退机制与地形合法性约束。
3. 数字分层 + 击杀反馈峰值。
4. 新增 SFX 变体时同步音频资源链路。

### PR-5.5-03 Topology + 决策可见化

1. 顺序连房升级为 MST + 回边。
2. 走廊形态多样化并保持可复现。
3. 事件/隐藏房/刷怪点联调。
4. 将关键分支决策显式写入 HUD/日志。

### PR-5.5-04 Heartbeat 表达层

1. 关键掉落触发“短脉冲反馈”（视觉或音频至少一项）。
2. 关键升级选择触发构筑提示（当前流派/潜在流派）。
3. 关键击杀（elite/boss）触发峰值反馈且可降级关闭。
4. 保证反馈优先级不遮挡生存信息。

### PR-5.5-05 Atmosphere/Readability

1. Idle 微动画、环境粒子、biome 氛围参数。
2. biome ambient loop 策略（独立或复用）。
3. 新手提示非阻断接入。
4. 资源缺失 fallback 与日志告警。

---

## 4. 资源治理要求

1. 视觉资源：`asset-plan -> manifest -> assets:validate`。
2. 音频资源：`audio-plan -> audio-manifest -> assets:audio:validate`。
3. 引用不存在资源属于阻塞问题，禁止带病合并。

---

## 5. 验证方案

### 5.1 自动化

```bash
pnpm --filter @blodex/core test
pnpm --filter @blodex/game-client test
pnpm --filter @blodex/game-client i18n:check
pnpm assets:audio:compile
pnpm assets:audio:validate
pnpm assets:validate
pnpm ci:check
```

新增专项：

1. 心跳事件写入完整性测试。
2. 关键反馈优先级不遮挡关键信息测试。

### 5.2 手动冒烟

1. 默认优先使用金手指快速推进到：窄走廊、高压战斗、Boss 层、多 biome 场景。
2. 验证路径失败率下降、战斗反馈稳定、拓扑差异可感知。
3. 验证关键掉落/升级/击杀/分支是否形成心跳时刻。
4. 验证 ambient loop 切换与缺失回退。
5. 补 1 轮非金手指流程复测。

---

## 6. 出口门禁

1. 体验主线（移动、战斗、拓扑、可读性）全部通过。
2. 无 P0/P1 可读性回归。
3. 视觉与音频资源链路校验通过。
4. Taste-T3 通过：每局至少 3 次心跳时刻。

---

## 7. 风险与止损

1. 风险：多系统体验改动叠加导致性能波动。  
   止损：启用分层开关，按功能逐项回退。
2. 风险：特效强化影响战斗读图。  
   止损：优先保留关键信息层，降低氛围强度。
3. 风险：心跳反馈泛滥导致疲劳。  
   止损：心跳事件去抖与限频，保持稀缺性。

---

## 8. 与 5.6 交接

进入 5.6 前确认：

1. 基础体验链路稳定。
2. 资源治理链路可复用。
3. 关键场景回归报告已归档。
4. 心跳统计可为“再来一局钩子”提供输入。
