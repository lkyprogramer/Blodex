# Phase 5 V2 - 5.0 语义基线冻结与契约统一

**阶段**: 5.0  
**优先级**: P0  
**目标类型**: 规则地基冻结

---

## 1. 阶段目标

在进入规则修复前，先统一“术语、单位、测试口径”，避免后续 PR 建立在错误前提上。

完成标准：

1. 关键数值单位有唯一来源（尤其 `critChance` 小数制）。
2. `ItemSpecialAffixKey` 的“展示语义 -> 结算语义”映射文档冻结。
3. 所有 F1~F8/A1/A2 都有可追踪的修复票据与验收项。

---

## 2. 输入问题

1. 当前文档和代码存在“显示语义”和“结算语义”脱节。
2. 阶段文档较多，执行口径容易漂移。
3. 规则修复和体验修复交叉，容易混改。

---

## 3. 范围与非目标

### 3.1 范围

1. 规则契约文档（单位、词缀、战斗事件、掉落层级）。
2. 基线测试清单与脚本落地。
3. Phase5 V2 阶段看板与 PR 模板。

### 3.2 非目标

1. 不做战斗公式改动。
2. 不做 AI 行为改动。
3. 不做视觉/音频内容制作。

---

## 5. 验证方案

### 5.1 自动化

```bash
pnpm --filter @blodex/core test
pnpm --filter @blodex/game-client test
pnpm check:architecture-budget
pnpm ci:check
```

### 5.2 手动冒烟

1. 使用金手指快速进入 Floor 3、Floor 5 与 Endless 入口。
2. 检查 HUD、日志、事件总线中的规则字段是否一致。
3. 不开金手指补 1 局 Normal 快速回归，确认主流程不受治理项影响。

## 6. 出口门禁

1. V2 契约文档齐全并冻结。

## 7. 风险与止损

1. 风险：文档冻结过慢导致研发停摆。  
   止损：契约最小化，只冻结 P0 项。
2. 风险：测试定义过宽导致难落地。  
   止损：先覆盖 F1/F2/F3/A1 直接相关断言。
