# Phase 5 V2 - 5.7 发布收口与 DoD 冻结（含 Taste 签署）

**阶段**: 5.7  
**优先级**: P0  
**目标类型**: 发布基线冻结

---

## 1. 阶段目标

将 5.0~5.6 的改动收敛为可发布基线，完成门禁冻结、证据归档、回滚演练，并完成 taste 签署。

完成标准：

1. 规则缺陷与架构缺陷全部关闭或有豁免。
2. 自动化、手动、性能、资源链路全绿。
3. taste 指标达标并有可复核证据。
4. 发布材料可复核。

---

## 2. 收口范围

1. DoD 冻结表与发布清单。
2. 全量回归矩阵执行。
3. 性能对比与风险豁免。
4. 回滚手册与演练记录。
5. taste 验收与签署记录。

---

## 3. 阶段任务

### PR-5.7-01 DoD 冻结与脚本对齐

1. 冻结最终预算目标：`1500/650/450`。
2. 脚本阈值与文档阈值严格一致。
3. F1~F8/A1/A2 状态逐条更新。

### PR-5.7-02 全量回归执行

1. 执行统一自动化门禁。
2. 执行手动阻塞场景矩阵。
3. 收敛 P0/P1 缺陷。

### PR-5.7-03 Taste 签署

1. 对样本 run 执行 taste 评分（优先金手指快速覆盖关键路径）。
2. 校验构筑分歧、战斗节奏、心跳时刻、再来一局钩子。
3. 输出签署结论与整改项。

### PR-5.7-04 发布资料与回滚演练

1. 生成 release readiness / regression / performance / rollback 文档。
2. 完成回滚演练（步骤、耗时、结果）。
3. 完成发布签署。

## 5. 手动阻塞矩阵

1. 默认优先使用金手指快速通关，覆盖 Normal/Hard/Boss/Endless、路径分支、事件商店、读档恢复关键节点。
2. 规则链路：词缀生效、AI 可走性、掉落分层、升级选择。
3. 体验链路：移动容错、战斗触感、拓扑差异、HUD 可读性。
4. taste 链路：构筑分歧、心跳时刻、结算建议可执行性。
5. 资源链路：视觉和音频缺失回退、清单一致性。
6. 补 1 轮非金手指全流程复测用于平衡体感确认。

## 7. 风险与止损

1. 风险：临发布发现规则回归。  
   止损：按阶段回滚（5.6 -> 5.5 -> 5.4）。
2. 风险：资源链路不一致。  
   止损：以清单校验为阻断，不接受人工口头豁免。
3. 风险：性能结论不可复核。  
   止损：无证据即不签署。
4. 风险：taste 指标达标但玩家体感仍弱。  
   止损：保留 Phase 6 首周专项体验迭代窗口。

---

## 8. Phase 5 Exit

1. 项目从“规则可信度不足 + 架构临界”进入“可持续演进”状态。
2. 规则、架构、体验、资源、发布五条链路闭环。
3. 签名体验具备可复核证据，可作为 Phase 6 输入。
