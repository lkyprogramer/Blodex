# Phase 5 V2 - 5.1 致命规则修复 I（物品与战斗语义 + 构筑骨架）

**阶段**: 5.1  
**优先级**: P0  
**对应问题**: F1, F2

---

## 1. 阶段目标

修复“词缀显示有但战斗不生效”和“`critChance` 单位错配”两大根因；同时把构筑乐趣的骨架接入主循环。

完成标准：

1. 词缀单位统一为可解释数值体系。
2. 10 个 special affix 全部有明确消费路径。
3. HUD、日志、战斗结果三者一致。
4. 至少 3 条 archetype 种子可在 Floor 3 前（允许金手指）形成可观测差异。

---

## 2. 问题定义

1. `ItemAffix` 与 `DerivedStats` 在 `critChance` 维度单位不一致，导致暴击率易失真。
2. `rolledSpecialAffixes` 主要停留展示层，核心链路消费不足。
3. 即便修复生效，若没有可落地构筑路径，玩家体感仍是“数值微调”。

---

## 3. 范围与非目标

### 3.1 范围

1. `packages/content/src/items.ts` affix 数值单位治理。
2. `packages/core/src/stats.ts`、`combat.ts`、`skill.ts` 接入 special affix 消费。
3. `apps/game-client` HUD 与日志展示口径修正。
4. archetype 种子映射（affix + skill + talent）与测试基线。

### 3.2 非目标

1. 不调整怪物 AI。
2. 不调整地图与拓扑。
3. 不做大规模技能重做。

---

## 4. 技术方案

### 4.1 单位统一方案

1. `critChance` 统一为小数（例如 1% = `0.01`）。
2. affix pool 中历史整数值全部换算为小数。
3. 添加 `critChance` 上下限断言与回归测试。

### 4.2 special affix 聚合层

新增：`packages/core/src/specialAffix.ts`

```ts
export interface SpecialAffixTotals {
  lifesteal: number;
  critDamage: number;
  aoeRadius: number;
  damageOverTime: number;
  thorns: number;
  healthRegen: number;
  dodgeChance: number;
  xpBonus: number;
  soulShardBonus: number;
  cooldownReduction: number;
}
```

提供：

1. `resolveSpecialAffixTotals(equipment)`
2. `clampSpecialAffixTotals(totals)`

### 4.3 消费接入策略

1. `combat.ts`：`lifesteal/critDamage/thorns/dodgeChance`。
2. `skill.ts`：`aoeRadius/damageOverTime/cooldownReduction`。
3. 掉落与收益结算：`xpBonus/soulShardBonus`。
4. 被动恢复：`healthRegen` 进入 tick 更新。

### 4.4 构筑种子冻结（Taste-T1 基线）

| Archetype | 核心词缀 | 核心技能/天赋 | 可观测结果 |
|---|---|---|---|
| 暴击速攻流 | `critChance + critDamage + attackSpeed` | 单体爆发技能 + 攻速天赋 | 短窗高 burst，Boss 压血快 |
| 荆棘续航流 | `thorns + healthRegen + lifesteal` | 生存向天赋 | 扛线稳定、反伤击杀显著 |
| 持续控场流 | `damageOverTime + aoeRadius + cooldownReduction` | 范围技能/控场技能 | 群怪清理效率高，节奏偏控场 |

备注：5.1 仅冻结种子与接入路径，5.3/5.6 再扩展完整构筑池。

1. 风险：词缀全面接入导致 TTK 失衡。

2. 风险：冷却缩减破坏技能节奏。  
   止损：设置全局下限（例如 `cooldownMs >= 300ms`）。
3. 风险：构筑种子被单一路线垄断。  
   止损：在 5.3 通过掉落/升级分歧补偿弱势路径。
