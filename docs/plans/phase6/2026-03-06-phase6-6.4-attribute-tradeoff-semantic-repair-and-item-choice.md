# Phase 6 - 6.4 属性 trade-off、语义修复与装备权衡

**阶段**: 6.4  
**优先级**: P1  
**对应问题**: DEX 全能 + `damageOverTime` 语义漂移 + 装备权衡不足 + negative affix 前置能力缺失

---

## 1. 阶段目标

修复现在“数值上能跑，但权衡感与语义不对”的问题，让选择真正变成 trade-off。

完成标准：

1. `STR / DEX / VIT / INT` 的职责边界清晰。
2. `damageOverTime` 不再处于语义漂移状态。
3. 至少一组 rare / unique 提供真实 trade-off。
4. HUD / tooltip / compare score 支持 signed affix。

---

## 2. 当前问题

1. DEX 当前覆盖面太广，后悔成本过低。
2. VIT 与 DEX 的生存/机动边界不干净。
3. `damageOverTime` 名称承诺与现实现义不一致。
4. 当前几乎全是正向 affix，装备更多是“更高数值”而不是“更强但更痛”。
5. compare / HUD / item score 还没准备好处理负值 affix。

---

## 3. 范围与非目标

### 3.1 范围

1. 属性贡献重排。
2. `damageOverTime` 语义修复。
3. signed affix 表现层。
4. curated trade-off item set。
5. item score / compare score 调整。

### 3.2 非目标

1. 不在 6.4 开启全局随机 negative affix。
2. 不在 6.4 直接重做所有物品 content。
3. 不在 6.4 调整 pacing 目标。
4. 不在 6.4 引入负词缀专属 icon、稀有度边框重做或大规模新装备美术；优先用文本、颜色和布局承载 trade-off。

---

## 4. 技术方案

### 4.1 属性职责重排

建议主方向：

1. `STR`：主攻 attack power / heavy skill scaling。
2. `DEX`：主攻 crit / attack speed / move speed。
3. `VIT`：主攻 max health / armor。
4. `INT`：主攻 max mana / skill efficiency / rotation uptime。

实施策略：

1. 不做“一刀切”硬改；优先降低 `DEX -> armor` 系数，再提高 `VIT -> armor` 权重。
2. 同步更新 level-up 文案与 delta preview。

### 4.2 DamageOverTime 语义修复

主路径：

1. 若阶段预算允许，落真正 DOT runtime。
2. 若阶段中期评估风险过高，则回退到语义重命名：
   - `skillBonusDamage`
   - 或同等明确名称
3. 二者必须二选一，禁止继续维持“名字是 DOT，效果是平伤”。

### 4.3 Signed Affix Presentation

1. `formatAffixValue` 支持正负值显示。
2. compare tooltip、compare prompt、item score 都要能处理负值。
3. signed affix 不允许只在数据层存在、UI 层失真。
4. 第一版只用现有 HUD 组件、颜色语义与排版表达正负项，不新增图标包。

### 4.4 Curated Trade-off Item Set

第一阶段不启用全局随机 negative affix，而是：

1. 先挑少量 rare / unique 做策划型 trade-off。
2. 为 slot 建立 identity：
   - weapon：输出 vs 机动 / 稳定
   - chest：耐久 vs 机动
   - boots：速度 vs 护甲
3. 示例方向：
   - 高伤双手牺牲移速
   - 重甲牺牲机动
   - 极速鞋牺牲护甲
4. 默认复用现有 item icon / equipment art，不把新武器或新护甲美术作为 Phase 6 阶段前置。

### 4.5 Item Score / Choice Score 修复

1. `calculateItemPowerScore()` 不能继续简单累加。
2. score 需要区分：
   - offense
   - defense
   - utility
3. compare UI 需展示“哪方面变强、哪方面变弱”，而不是只给单一分数。
