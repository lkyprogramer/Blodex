# Phase 6 - 6.3 心跳时刻表达层与装备比较反馈

**阶段**: 6.3  
**优先级**: P1  
**对应问题**: rare / build / boss 峰值缺表现层 + 装备比较仅依赖 hover tooltip

---

## 1. 阶段目标

为已有规则补齐玩家真正能看见、听见、记住的反馈层，并把装备比较从被动 hover 升级为主动时刻。

完成标准：

1. rare / unique 掉落具备专属反馈 cue。
2. build formed 具备实时正反馈。
3. boss / elite / near-death 具备高压峰值反馈。
4. synergy 激活具备实时反馈。
5. 装备比较从“hover tooltip”升级为“拾取 / 购买 / 升级后的主动比较”。
6. 每局至少 4 个可见心跳时刻。

---

## 2. 当前问题

1. 关键掉落现在主要是逻辑记账，没有专属表达层峰值。
2. build identity 已可记录，但没有 run 内正反馈消费者。
3. 装备比较只在鼠标悬浮时出现，不符合关键抉择时机。
4. 当前反馈路由缺少 `rare_drop / build_formed / boss_reward` 这类高价值事件。
5. 当前 `SFXSystem` 没有 rare drop / build formed / boss reward / equipment compare 的专属音频 key，现有音频无法满足“专属 cue”的目标。
6. 当前 `VFXSystem` 已具备 flash / shake / floating text / ellipse 等 code-driven 能力，Phase 6 的视觉基线不一定需要新增图片资产。
7. `synergyRuntime` 已在后台计算激活结果，但玩家侧没有“synergy activated”反馈。

---

## 3. 范围与非目标

### 3.1 范围

1. rare / unique 掉落表现层。
2. build formed 实时反馈。
3. boss / elite / near-death 峰值反馈。
4. `FeedbackRouterV2`。
5. `EquipmentComparePrompt`。
6. `synergy` 激活反馈。

### 3.2 非目标

1. 不在 6.3 重新定义 spike 规则。
2. 不在 6.3 改动物品内容池。
3. 不在 6.3 重写完整 HUD。

---

## 4. 技术方案

### 4.1 Rare / Unique Drop Cue

1. 为 rare / unique 掉落接入：
   - 屏幕 flash
   - hit-stop 或短时 slow
   - 专属 SFX
2. 反馈必须限频，避免连续掉落时噪音失控。
3. cue 与 `power_spike`、`rare_drop_presented` 事件对齐。

### 4.2 Build Formed Runtime Consumer

1. 为 `TasteRuntimePortHub` 的 build tag / threshold 增加消费者。
2. 达到条件时触发：
   - toast
   - HUD 微提示
   - 日志条目
3. 仅在第一次达成或明显升级时触发，避免刷屏。
4. `build formed` 与 `synergy activated` 共享反馈预算，避免同一时刻双重刷屏。

### 4.3 高压战斗峰值反馈

1. elite / boss 关键击杀
2. near-death reversal
3. boss reward close

要求：

1. 不遮挡生存信息。
2. 优先级高于氛围层，低于死亡 / 警告层。

### 4.4 EquipmentComparePrompt

1. 触发时机：
   - 拾取 rare / unique
   - 商店高价值购买
   - 技能/路线导致的关键件获得
2. 支持动作：
   - 立即装备
   - 稍后查看
   - 忽略
3. compare prompt 与 tooltip 共享同一份 delta 数据，不重复算两套逻辑。

### 4.5 FeedbackRouterV2

扩展路由覆盖：

1. `loot:rare_drop`
2. `build:formed`
3. `power_spike`
4. `boss:reward`
5. `synergy:activated`

### 4.6 资源增量与复用策略

1. 本阶段的音频资源增量是硬需求；最小集合至少覆盖：
   - `ui_loot_rare_drop_01`
   - `ui_build_formed_01`
   - `ui_boss_reward_open_01`
   - `ui_equipment_compare_open_01`
2. 上述音频 id 对应的事件键建议分别绑定：
   - `loot:rare_drop`
   - `build:formed`
   - `boss:reward`
   - `equipment:compare`
3. 视觉层默认优先复用现有 `VFXSystem` 的 camera flash、shake、tint、floating text、ellipse 与现有 HUD/prompt 样式。
4. 如需进一步拉开区分度，可选新增少量视觉资源，但不作为 6.3 P0 前置：
   - `vfx_loot_rare_pulse_01`
   - `ui_build_tag_badge_01`
   - `ui_compare_prompt_frame_01`
5. 任何视觉或音频增量都必须同步更新：
   - `assets/source-prompts/asset-plan.yaml`
   - `assets/generated/manifest.json`
   - `assets/source-prompts/audio-plan.yaml`
   - `assets/generated/audio-manifest.json`
   - 对应 `public/generated` 或 `public/audio` 运行目录

4. 资源清单与运行时 key 一致性测试。

### 6.2 手动冒烟

1. 默认优先使用金手指刷 rare、unique、boss victory、merchant 高价值商品。
2. 检查 rare / build / boss cue 是否都能稳定触发。
3. 检查 compare prompt 是否在正确时机出现且不打断主流程。
4. 补 1 局非金手指验证心跳时刻密度与疲劳度。
5. 检查新增资源缺失时是否有可接受 fallback，且日志不出现静默丢 key。
6. 至少触发 1 次 synergy 激活并确认玩家能知道“触发了什么、改变了什么”。

## 7. 出口门禁

1. rare / build / boss 三类峰值均有专属 cue。
2. 每局至少 4 个可见心跳时刻。
3. 装备比较不再只依赖 hover tooltip。
4. 反馈层不遮挡关键生存信息。
5. 本阶段新增的视觉/音频资源全部完成 manifest 校验。
6. synergy 激活在玩家侧有明确 cue。

---

## 8. 风险与止损

1. 风险：反馈层过量导致噪音。  
   止损：分级、限频、可降级关闭。
2. 风险：compare prompt 打断流畅性。  
   止损：只在高价值节点触发，普通掉落继续走轻量提示。
3. 风险：视觉强化影响读图。  
   止损：优先保留血量、危险区、Boss telegraph。
4. 风险：资源引用、manifest 与运行时 key 不一致。  
   止损：把 `assets:audio:validate` 与 `assets:validate` 作为本阶段硬门禁。
5. 风险：反馈层接线继续把 `DungeonScene` / `HudContainer` 推高。  
   止损：优先抽离到 `feedback/*` 与独立 component；若本阶段仍触发历史预算债，必须用显式 follow-up 冻结 debt ceiling，而不是放宽目标预算。
