# Phase 6 - 6.1 玩家代理权、战斗节奏与职业起步深度

**阶段**: 6.1  
**优先级**: P0  
**对应问题**: 关键选择缺代理权 + 自动攻击主导战斗 + mana/CD 双重门槛 + arcanist 起步深度不足 + buff/debuff runtime 空转 + damageType 身份不足

---

## 1. 阶段目标

让“每层至少一次关键选择”从预算兜底升级为玩家可感知的高价值选择，并把战斗从“主要看自动攻击”拉回到“技能稳定参与循环”。

完成标准：

1. 升级技能从后台自动发放改成玩家可见的 `3 选 1`。
2. 属性选择具备 `delta preview`，不再只是抽象加点。
3. 常规战斗中，玩家平均每 `2.5s~4s` 有一次主动输入窗口。
4. `skill casts / 30s`、`auto-attack damage share`、`mana dry window`、`average no-input gap` 相比 6.0 baseline 明显改善。
5. `arcanist` 的起步深度不再明显低于 warrior / ranger。
6. `war_cry / shadow_step / frost_nova` 的效果在 runtime 中真实生效。
7. `physical / arcane` 具备最小策略差异，`arcanist` 获得可感知身份。

---

## 2. 当前问题

1. 升级技能现在是后台抽 3 取 1，玩家没有代理权。
2. 自动攻击是常规战斗主轴，技能更多是补充动作。
3. 当前没有常驻 mana 自动回复；法力恢复过度依赖击杀、药水和事件。
4. 大量技能 CD 偏长，导致“有技能”不等于“技能进入循环”。
5. `arcanist` 只有 2 个默认可用技能，3 个关键技能被 blueprint 锁住。
6. `resolveSkill()` 已返回 `buffsApplied`，但 runtime 没有把它写回状态，也没有统一 buff 定义表。
7. `damageType` 虽然存在于 skill def 与 combat event 中，但 physical / arcane 仍无规则差异。

---

## 3. 范围与非目标

### 3.1 范围

1. 升级技能选择面板。
2. 升级属性 `delta preview`。
3. `floor choice budget` 从“面板出现”升级为“高价值选择出现”。
4. mana economy 与核心技能 CD 重排。
5. `arcanist` 起步技能池补齐。
6. 战斗节奏 telemetry 与门禁。
7. `buff/debuff` runtime 闭环。
8. `damageType` 最小差异化。

### 3.2 非目标

1. 不在 6.1 完整重写战斗系统。
2. 不在 6.1 引入完整主动闪避 / 格挡系统。
3. 不在 6.1 做 rare / unique 表现层。
4. 不在 6.1 调整装备词缀语义。
5. 不在 6.1 引入独立美术/音频资源包；若后续需要 choice open/confirm cue，统一并入 6.3。

---

## 4. 技术方案

### 4.1 升级技能显式选择

1. 将现有后台 `offerLevelupSkill()` 改为：
   - 构建候选池
   - 输出玩家可见 `3 选 1`
   - 玩家选择后再写入技能槽
2. 候选逻辑仍保留 `strongestStat + ownedSkillIds` 权重，但不再自动吞掉选择。
3. 允许“已有技能升级”与“新技能获得”同时出现在候选中。
4. 默认复用现有 skill icon 与通用面板样式，不把新 icon / 新面板美术作为前置。

### 4.2 升级属性 Delta Preview

1. 属性面板展示选择后的即时差异：
   - attack power / crit / attack speed
   - max health / armor
   - 关键技能收益变化
2. 所有 preview 基于当前装备、当前技能、当前主武器实时计算。
3. preview 仅做解释，不改变规则结算。
4. preview 第一版优先采用文本、颜色和现有 formatter 表达，不新增专属图标包。

### 4.3 Combat Tempo Rebaseline

1. 引入基础 mana 自动回复。
2. 重排核心技能 CD：
   - 常规技能进入 `2s~5s`
   - 高影响技能保留长 CD，但不主导全部职业
3. 设定四个主指标：
   - `skill casts / 30s`
   - `auto-attack damage share`
   - `mana dry window`
   - `boss fight minimum usable skill count`
4. Phase 6 不承诺主动防御系统重写；若仅靠数值调优仍不达标，再将 dodge/parry/dash 纳入后续输入。

### 4.4 Arcanist 起步深度最小成本方案

默认优先方案：

1. 移除 `chain_lightning / spirit_burst / rift_step` 的基础 `unlockCondition`。
2. 将原 blueprint 改为：
   - 高级变体
   - 数值强化
   - 额外修饰
3. 仅当此方案验证失败时，再考虑新增无 blueprint 门槛的新技能。

### 4.5 BuffRuntimeClosure

1. 定义权威 `BuffDefMap / BuffRuntimeRegistry`，不再让 buff 定义只存在于测试桩里。
2. 让 `resolveSkill().buffsApplied` 真正落到 `player.activeBuffs`，并为 monster runtime 增加 debuff 承载。
3. 统一 apply / update / expire 流程，使属性、移速、guaranteed crit 能通过同一条运行时链路生效。
4. 修正 AOE debuff 目标模型，使 `frost_nova` 对所有命中目标生效，而不是只落到第一个目标。

### 4.6 DamageTypeResolverV1

1. 引入最小 `DamageTypeResolverV1` 挂点，避免 `damageType` 只停留在日志层。
2. Phase 6 只承诺 `physical / arcane` 的最小差异化：
   - `physical` 继续主受 armor 影响；
   - `arcane` 走独立 mitigation hook 或部分 armor bypass。
3. 不在 6.1 扩展 `fire / cold / lightning` 或完整抗性表。
4. telemetry 与日志需要能输出 damage type share / mitigation 差异。

## 6. 验证方案

### 6.1 自动化

```bash
pnpm --filter @blodex/core test
pnpm --filter @blodex/game-client test
pnpm ci:check
```

新增重点：

1. 升级技能显式选择测试。
2. 属性 preview 正确性测试。
3. mana regen 与 skill availability 测试。
4. `arcanist` 起步技能池可用性测试。
5. rhythm 指标回归测试。
6. buff apply / expire 与属性消费测试。
7. `frost_nova` AOE debuff 覆盖测试。
8. `physical / arcane` 最小差异化契约测试。

### 6.2 手动冒烟

1. 默认优先使用金手指推进到多次升级场景。
2. 对比 6.0 baseline 与 6.1 改造后的：
   - 平均无主动输入间隔
   - 技能释放频率
   - 自动攻击 / 技能伤害占比
3. 三职业各跑至少 1 局，验证起步深度差异。
4. 补 1 局非金手指验证实际战斗节奏。
5. `war_cry / shadow_step / frost_nova` 各至少触发 1 次，确认效果真实生效。
6. 至少做 1 组 `physical / arcane` 对比，确认 `arcanist` 的伤害身份不再与 warrior 完全同轨。

## 7. 出口门禁

1. 每层至少有 1 次 `player_facing_choice`。
2. 升级技能成长不再由后台自动分配。
3. 常规战斗节奏合同达标。
4. `arcanist` 的开局深度不再明显低于 warrior / ranger。
5. 现有 buff/debuff 技能不再停留在 `buffId`，而是 runtime 真正生效。
6. `physical / arcane` 差异对玩家可感知。

---

## 8. 风险与止损

1. 风险：mana regen + CD 缩短导致数值失衡。  
   止损：先以 telemetry 验证节奏改善，再做伤害回压。
2. 风险：显式 skill offer 打断节奏。  
   止损：只在安全窗口打开，不与 boss 高压过程重叠。
3. 风险：`arcanist` 直接解锁 3 技能后职业爆强。  
   止损：先做可用性对齐，再在 6.5 回收强度。
