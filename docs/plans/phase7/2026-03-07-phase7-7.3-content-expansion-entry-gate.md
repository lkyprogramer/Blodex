# Phase 7 - 7.3 内容扩展入口门

**阶段**: `7.3`  
**优先级**: `P2 Gate`  
**对应问题**: 在技术债与 Phase 6 未正式签署前，禁止恢复内容扩展

---

## 1. 阶段目标

这不是玩法开发阶段，而是内容扩展的正式解锁门。

完成标准：

1. `7.0A ~ 7.1` 已全部通过。
2. Phase 6 已正式签署。
3. architecture budget 不再依赖核心大文件 debt ceiling。
4. save/resume / evidence / sign-off 基础设施稳定。

---

## 2. 当前问题

1. 如果没有这个 gate，团队会自然滑回“先加 Boss / 内容，再回头修债”的老路。
2. 内容扩展一旦先行，Scene/UI/save/evidence 债会再次变得不可拆。

---

## 3. 范围与非目标

### 3.1 范围

1. gate checklist
2. 文档与 CI 门禁
3. 进入内容扩展阶段的准入判断

### 3.2 非目标

1. 不在本阶段直接实现内容扩展。
2. 不在本阶段直接生产新的 gameplay 美术或音频资源。

---

## 4. 技术方案

### 4.1 解锁条件

只有以下全部满足，才允许进入 `7.4 ~ 7.8`：

1. `7.0A` 完成
2. `7.0B` 完成
3. `7.0C` 完成
4. `7.1` 完成
5. Phase 6 release readiness 为 `Signed`
6. regression matrix 无阻塞 pending

### 4.2 执行约束

1. 在 gate 未通过前，任何 `7.4+` PR 一律拒绝。
2. backlog 文档只保留为参考，不构成执行许可。

### 4.3 资源入口门

进入 `7.4 ~ 7.8` 前，资源侧也必须同时满足：

1. 视觉方向已明确绑定 [art-style-bible.md](/Users/luo/Documents/github/Blodex/docs/art-style-bible.md)，不得再出现独立风格分支。
2. `assets/source-prompts/asset-plan.yaml` 中已为下一阶段补齐最小发货资源清单：
   - `minimum_ship_set`
   - `optional_enhancement_set`
   - `placeholder_policy`
3. `assets/source-prompts/audio-plan.yaml` 中已为下一阶段补齐最小音频 cue 清单与命名规范。
4. 资源生产顺序固定为：
   - 先冻结 prompt / plan
   - 再生成 manifest entry
   - 最后进入运行时接线
5. 在资源入口门通过前，允许使用占位图与既有 cue 做玩法验证，但不允许把未登记资源直接接入主线阶段 PR。
6. 美术资源生成的额外前提是：用户已显式提供 Gemini Key；在此之前，`asset-plan` 只能进入“待生成”状态，不能直接执行生成。

### 4.4 阶段职责

本阶段必须把后续资源工作拆成两类：

1. `Minimum Ship Set`
   - 每个阶段能完成玩法闭环所必需的 sprite / icon / cue / badge / node marker。
2. `Optional Enhancement Set`
   - 只增强表现力、不阻断玩法闭环的 portrait / phase variant / extra flourish / alternate cue。

只有 `Minimum Ship Set` 允许成为阶段出口门禁；`Optional Enhancement Set` 只能作为加分项。

---

## 5. 验证方案

1. 检查 architecture budget
2. 检查 release readiness
3. 检查 sign-off checklist
4. 检查 artifact index
5. 检查 `asset-plan.yaml` / `audio-plan.yaml` 是否存在下一阶段的最小资源清单
6. 检查资源命名、manifest 入口与 style bible 引用是否完整
7. 检查所有待生成美术资源是否都标注了 `Gemini Key Required`，且没有被误记为已生成

---

## 6. 出口门禁

1. 内容扩展可以正式启动。
2. `phase7 roadmap` 与 `carryover backlog` 不再有执行歧义。

---

## 7. 风险与止损

1. 风险：gate 流于口头规则。  
   止损：必须固化成文档与 CI/PR 审查规则。

---

## 8. 与 7.4 交接

进入 `7.4` 前必须确认：

1. 本阶段的 gate checklist 已被显式标记为通过。
2. `7.4` 的 Boss 资源最小发货集已经在 `asset-plan` 与 `audio-plan` 中冻结。
3. 若尚未提供 Gemini Key，`7.4` 只能推进 Boss 资源清单、命名、占位图与运行时接线，不得直接进入批量美术生成。
