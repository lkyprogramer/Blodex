# Phase 7 - 7.8 长 Run 与楼层扩展

**阶段**: `7.8`  
**优先级**: `P3`  
**前置条件**: `7.3` gate 已通过  
**对应问题**: 当前 `5` 层短 run 的构筑演化长度与内容承载上限已接近饱和

---

## 1. 阶段目标

把当前 `5` 层短 run 扩展为 `8~10` 层，并加入中期节点，使更长线构筑真正成立。

完成标准：

1. 楼层从 `5` 扩展到 `8~10`
2. 中期节点成立
3. pacing model 与 reward curve 重新平衡
4. save snapshot 与 summary 模型适配更长 run

---

1. `5` 层结构限制了中期 build 演化。
2. 更长的 reward curve、shop/event cadence、mid-run decision 还没有承载空间。

1. 风险：run 变长但中期节点价值不足。

   止损：只复用 `7.0B` 和 `7.0C` 建好的 pipeline/registry。
