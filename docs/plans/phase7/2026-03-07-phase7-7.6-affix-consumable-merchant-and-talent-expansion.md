# Phase 7 - 7.6 Affix、Consumable、Merchant 与 Talent 扩充

**阶段**: `7.6`  
**优先级**: `P2`  
**前置条件**: `7.3` gate 已通过  
**对应问题**: 中层内容密度不足，run 差异性依然主要靠装备与路线分歧

---

## 1. 阶段目标

扩充中层内容密度，让普通房间、商店、消耗品和天赋投入形成更高频的 replayability。

完成标准：

1. monster affix 扩到 `8+`
2. consumable 内容池正式激活
3. merchant 从纯随机事件增强为更稳定遭遇
4. talent 从开/关两态扩到更有投入感的 rank 模型

---

## 2. 当前问题

1. 普通战斗中层差异还不够强。
2. consumable 类型存在，但内容密度不足。
3. merchant 稳定性不足。
4. talent 投入感不强。

---

## 3. 范围与非目标

### 3.1 范围

1. affix 批次一
2. consumable enablement
3. merchant stabilization
4. talent rank expansion

### 3.2 非目标

1. 不在本阶段扩完整元素体系。
2. 不在本阶段扩长 run。

---

## 4. 技术方案

### 4.1 Monster Affix Batch One

原则：

1. 优先行为差异，不优先数值词缀。
2. affix 必须有可感知行为与反馈。

### 4.2 Consumable Enablement

原则：

1. 每类 consumable 必须有明确用途与 UI 表达。
2. 不做无意义的平铺药水池。

### 4.3 Merchant Stabilization

方向：

1. 提高商店的稳定出现概率或独立房间化。
2. 商店 compare / value gating 继续复用现有 compare 系统。

### 4.4 Talent Rank Expansion

方向：

1. 从 binary unlock 改成 rank model
2. rank 带来明确成长曲线与选择压力

### 4.5 美术与音频方案

#### Monster Affix

1. 默认优先复用现有怪物底图，通过：
   - tint
   - overlay glyph
   - telegraph variation
   - 小型 VFX anchor
   表达 affix 身份
2. 只有行为差异无法靠现有层表达时，才新增专属 affix overlay 资产。
3. 风格必须遵循 [art-style-bible.md](/Users/luo/Documents/github/Blodex/docs/art-style-bible.md)，禁止高饱和状态特效。

#### Consumable

1. 最小视觉资源集：
   - `item_consumable_health_*`
   - `item_consumable_mana_*`
   - `item_consumable_scroll_*`
   - `item_consumable_elixir_*`
2. icon 必须保持 painterly dark fantasy 的单体可读性，背景透明，避免现代药瓶或高亮玻璃质感。

#### Merchant

1. 若升级为独立节点/房间，最小视觉资源集：
   - `merchant_node_marker_01`
   - `merchant_room_sign_01`
   - `merchant_portrait_01`（可选）
2. 音频最小增量：
   - `ui_merchant_special_open_01`
   - `ui_merchant_special_buy_01`

#### Talent

1. rank 扩展优先复用现有 icon 系统。
2. 只有当 rank 表达无法靠文本/数字解决时，再补充：
   - `ui_talent_rank_pip_01`
   - `ui_talent_rank_glow_01`

#### 资产治理

1. affix / consumable / merchant / talent 的资源必须分批写入 `asset-plan` 与 `audio-plan`。
2. 每一批都先定义最小上线集，再定义可选增强集。
3. 任何新增 icon / marker / portrait / overlay 的实际美术生成，都必须等待用户提供 Gemini Key。
4. 在 Gemini Key 未提供前，本阶段只允许完成资源清单、prompt、命名和占位接线，不允许直接运行图片生成。
