# Phase 7 - 7.5 Boss 内容批次一

**阶段**: `7.5`  
**优先级**: `P2`  
**前置条件**: `7.4` 已完成  
**对应问题**: 缺少 Boss 广度，5 层 run 的高潮重复感过强

---

## 1. 阶段目标

新增 `2~3` 个 Boss，并确保它们在机制、节奏、奖励和反馈语义上与现有骨王形成清晰区分。

完成标准：

1. 至少新增 `2` 个可上线 Boss。
2. 每个 Boss 具备独立 phase、attack pattern、telegraph 与 reward identity。
3. compare / feedback / summary / reward 链路不需要为每个 Boss 再加场景分支。

---

## 2. 当前问题

1. 只有一个 story Boss，重复游玩新鲜感不足。
2. Boss 机制与 build identity 的互动还不够多样。

---

## 3. 范围与非目标

2. 新 Boss `B`

### 3.2 非目标

---

## 4. 技术方案

### 4.1 Boss 设计原则

每个 Boss 至少提供一种差异：

1. 位置压迫
2. 召唤物处理
3. 节奏窗体打断
4. 对 build identity 的偏好测试

### 4.2 奖励身份

每个 Boss 的 reward 需要有明显 identity：

1. 偏 offense
2. 偏 defense
3. 偏 utility / branch / set-up

### 4.3 内容接线

1. 所有内容通过 `BossRegistry` 接入。
2. 不允许直接在 scene/boss runtime 中写死新 boss 分支。

### 4.4 美术方案

#### 风格约束

全部 Boss 资产必须遵循 [art-style-bible.md](/Users/luo/Documents/github/Blodex/docs/art-style-bible.md)：

1. `painterly-dark-fantasy-v1`
2. sombre medieval-cathedral gloom
3. clear isometric silhouette
4. restrained highlight, no neon, no bloom-heavy haze

#### 最小视觉资源集（每个 Boss）

1. `boss_sprite_<id>`
2. `boss_portrait_<id>`
3. `boss_phase_variant_<id>_01`（如 phase 变化需要）
4. `boss_telegraph_circle_<id>`（如需独立 telegraph 形状）
5. `boss_reward_badge_<id>`（可选）

#### 资源建议

1. Boss A：强调重甲 / 压迫 / 金属与石质材料
2. Boss B：强调异教 / 施法 / 低饱和冷暖对比
3. Boss C：强调召唤 / 腐化 / 骨与布料材质

#### 产出流程

1. 先补 `asset-plan.yaml`
2. 再生成/导入到 `manifest`
3. 再接 runtime
4. 资源不到位时，先用现有 telegraph 与 portrait 占位，不阻塞管线验证
5. 任何 Boss 美术资源生成都必须在用户提供 Gemini Key 后执行；在此之前，文档、prompt、manifest 占位可以先做，但不能直接跑生成。

### 4.5 音频方案

#### 最小音频资源集（每个 Boss）

1. `sfx_boss_<id>_intro_01`
2. `sfx_boss_<id>_phase_change_01`
3. `sfx_boss_<id>_signature_attack_01`
4. `sfx_boss_<id>_defeat_01`
5. `ui_boss_<id>_reward_open_01`（可选，若 reward identity 需要）

#### 风格要求

1. 延续 `grim-fantasy-audio-v1`
2. 偏石质、骨质、金属、祭祀、低频压迫
3. 避免亮电子、科幻感、过度混响

#### 生成前置

1. `boss_sprite_<id>`、`boss_portrait_<id>`、`boss_phase_variant_<id>_01` 等视觉资产的实际生产，必须等待用户提供 Gemini Key。
2. 在 Key 未提供前，阶段只能完成玩法、命名、资源位和占位图接线，不得把 Boss 美术生成纳入默认执行步骤。
