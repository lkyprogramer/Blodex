### 4.4 美术方案

#### 元素体系视觉方向

所有元素视觉必须在 [art-style-bible.md](/Users/luo/Documents/github/Blodex/docs/art-style-bible.md) 的限制下做区分：

1. `physical`
   - 几乎无魔法辉光
   - 以金属/冲击/碎石感为主
2. `arcane`
   - 冷色、克制、符文感
   - 不允许科幻紫电视觉
3. `fire`
   - ember orange / soot black
   - 控制亮度，不做屏幕泛白火海
4. `cold`
   - iron blue / frost white
   - 偏结晶与寒雾，不做高亮霓虹蓝
5. `lightning`
   - pale gold / cold white
   - 控制闪烁时长，确保 gameplay readability

#### 最小视觉资源集

1. `fx_element_fire_*`
2. `fx_element_cold_*`
3. `fx_element_lightning_*`
4. `ui_element_icon_*`
5. `ui_set_badge_*`
6. `ui_set_partial_bonus_frame_*`（可选）

#### 生成前置

1. 上述元素与 set 的新增视觉资源，只有在用户提供 Gemini Key 后才允许执行生成。
2. 在 Gemini Key 未提供前，只能完成 style prompt、resource spec、manifest 占位和运行时接线准备。

### 4.5 音频方案

#### 元素音频最小集合

1. `sfx_element_fire_hit_01`
2. `sfx_element_cold_hit_01`
3. `sfx_element_lightning_hit_01`
4. `ui_set_activated_01`
5. `ui_set_upgraded_01`（可选）

#### 风格要求

1. 继续遵循 `grim-fantasy-audio-v1`
2. fire 偏 ember/roar
3. cold 偏 brittle/stone frost
4. lightning 偏 sacred crack，不偏科幻 synth

#### Set 表达原则

1. set feedback 不能喧宾夺主，必须低于 rare drop / boss reward cue。
2. set icon 与 set badge 必须强调 medieval relic / oath / sigil 感，不走高饱和 MMO 套装风格。
